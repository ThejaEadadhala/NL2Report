# NL2Report — TPC-H Data Processing Project
**Week 3 — Real TPC-H Dataset, PostgreSQL Backend**

---

## What this project does

Generates real TPC-H benchmark data at Scale Factor 1 (~1GB, 6 million rows),
cleans and transforms all 8 tables, loads them into PostgreSQL, and provides
a query/filter tool to slice the data.

---

## Project structure

```
NL2Report/
  ├── raw/                  # TPC-H CSVs generated by generate_data.py (git ignored)
  ├── processed/            # cleaned CSVs output by tpch_pipeline.py (git ignored)
  ├── venv/                 # Python virtual environment (git ignored)
  ├── generate_data.py      # STEP 3 — generates real TPC-H SF=1 data using DuckDB
  ├── tpch_pipeline.py      # STEP 5 — cleans data and loads into PostgreSQL
  ├── tpch_filter.py        # STEP 6 — query/filter data from PostgreSQL
  ├── schema.sql            # PostgreSQL table definitions (8 tables)
  ├── constraints.sql       # primary keys, foreign keys, indexes
  ├── requirements.txt      # Python dependencies
  └── README.md             # this file
```

## Setup — follow these steps in order

### Step 1 — Clone the repository

```bash
git clone https://github.com/ThejaEadadhala/NL2Report.git
cd NL2Report
```

---

### Step 2 — Create and activate virtual environment

```bash
python -m venv venv
venv\Scripts\activate
```

You should see `(venv)` at the start of your terminal line.

> Every time you open a new terminal for this project, run `venv\Scripts\activate` first.

---

### Step 3 — Install Python dependencies

```bash
pip install -r requirements.txt
```

---

### Step 4 — Generate TPC-H data

```bash
python generate_data.py
```

This generates all 8 TPC-H tables as CSV files in the `raw/` folder.
Takes about 2–5 minutes.

Expected output:
```
exported customer.csv  (150,000 rows)
exported lineitem.csv  (6,001,215 rows)
exported nation.csv    (25 rows)
exported orders.csv    (1,500,000 rows)
exported part.csv      (200,000 rows)
exported partsupp.csv  (800,000 rows)
exported region.csv    (5 rows)
exported supplier.csv  (10,000 rows)
Done! All 8 tables saved to raw/
```

---

### Step 5 — Create PostgreSQL database and schema

```bash
psql -U postgres -c "CREATE DATABASE tpch;"
psql -U postgres -d tpch -f schema.sql
```

Enter your PostgreSQL password when asked.

---

### Step 6 — Update your PostgreSQL password in the pipeline files

Open `tpch_pipeline.py` and `tpch_filter.py` and change this line:

```python
PG_PASSWORD = "theja"   # change this to YOUR PostgreSQL password
```

---

### Step 7 — Run the pipeline

```bash
python tpch_pipeline.py
```

This reads from `raw/`, cleans all tables, saves to `processed/`, and loads into PostgreSQL.
Takes about 15–20 minutes (lineitem has 6 million rows).

Expected output at the end:
```
Table                Rows loaded
────────────────────────────────────────
region                           5
nation                          25
supplier                    10,000
part                       200,000
partsupp                   800,000
customer                   150,000
orders                   1,500,000
lineitem                 6,001,215
```

---

### Step 8 — Apply constraints

```bash
psql -U postgres -d tpch -f constraints.sql
```

Adds primary keys, foreign keys and indexes for faster querying.

---

### Step 9 — Verify everything is loaded

```bash
psql -U postgres -d tpch -c "\dt"
```

You should see all 8 tables listed.

---

### Step 10 — Test the filter tool

```bash
python tpch_filter.py
```

Runs example queries against PostgreSQL and prints results.

## Using the filter tool in your own code

```python
from tpch_filter import connect, filter_orders, filter_lineitem, query

engine = connect()

# Filter urgent orders in 1995 over $50,000
df = filter_orders(engine, priority="1-URGENT",
                   date_from="1995-01-01", min_price=50000)

# Air-shipped items with discount under 5%
df2 = filter_lineitem(engine, ship_mode="AIR", max_discount=0.05)

# Custom SQL query
df3 = query(engine, """
    SELECT l_shipmode,
           COUNT(*) AS line_count,
           ROUND(SUM(l_extendedprice * (1 - l_discount))::numeric, 2) AS revenue
    FROM lineitem
    GROUP BY l_shipmode
    ORDER BY revenue DESC
""")
```

## Common errors

| Error | Fix |
|-------|-----|
| `psql not recognized` | Add `C:\Program Files\PostgreSQL\18\bin` to system PATH, restart VS Code |
| `ModuleNotFoundError` | Run `venv\Scripts\activate` then `pip install -r requirements.txt` |
| `password authentication failed` | Update `PG_PASSWORD` in `tpch_pipeline.py` and `tpch_filter.py` |
| `could not connect to PostgreSQL` | Make sure PostgreSQL service is running — search "Services" in Start menu |
